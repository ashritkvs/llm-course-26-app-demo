Build a polished full-stack web app called UniFlow: AI-Powered Unified Course Assistant.

Context:
This is for my AMS 691 App Building with AI/LLM course demo submission. The project should be a working prototype for a course showcase website. It should be impressive, demo-ready, and honest: it should use sample data and paste-input data instead of requiring real Brightspace, Discord, Google Drive, or professor feedback form integrations.

Use:
- Next.js
- TypeScript
- Tailwind CSS
- Clean dark dashboard UI
- Teal/blue accent colors
- Responsive layout
- Professional SaaS-style interface

Main idea:
Students manage course information across multiple scattered platforms:
- Brightspace announcements
- Discord discussions
- Google Drive documents
- Quizzes / assignments
- Peer presentation feedback forms

UniFlow should unify these scattered updates into one AI-powered dashboard that summarizes announcements, extracts tasks/deadlines, answers course-related questions, and helps students write structured peer feedback after class presentations.

Important:
Do NOT require real Brightspace, Discord, Google Drive, or Google Forms API integration.
Build a complete working prototype using:
- realistic demo/sample data
- user paste input
- localStorage for persistence
- fallback demo AI output when no API key is available

Core pages:

1. Dashboard
Create a main dashboard showing:
- weekly priority summary
- missed deadlines
- upcoming deadlines
- urgent tasks
- platform/source breakdown
- recent course updates
- AI-generated weekly action plan
- quick button: “What should I complete this week?”

2. Sources Page
Create a page where users can manage course sources:
- Brightspace card

- Presentation Feedback card

Each source should show:
- source name

- button to load realistic sample AMS-style course data

3. Tasks Page

- task title

- due date
- priority
- status

4. AI Assistant Page
Create a chat-style assistant page.
The user should be able to ask:
- “What should I complete this week?”
- “What deadlines am I missing?”
- “Summarize my course updates.”
- “What do I need for the app demo submission?”
- “Which presentations still need feedback?”
- “Help me write feedback for this student’s presentation.”

The assistant should answer using the loaded sample/pasted course data.
If no real LLM API key is configured, return high-quality realistic demo responses.

5. Presentation Feedback Page
Add a dedicated page called “Feedback Helper” because in this course we also need to submit reviews after each student presentation.

This page should help students create clear feedback after watching classmates’ presentations.

Features:
- Form fields:
  - presenter name
  - project/app title
  - what was best about the presentation
  - one improvement suggestion
  - clarity rating from 1 to 5
  - demo quality rating from 1 to 5
  - usefulness/impact rating from 1 to 5
  - optional notes

- AI button: “Polish Feedback”
  It should convert rough notes into a concise respectful review.

- Output format:
  Best part:
  Improvement suggestion:
  Overall feedback:

- Add copy-to-clipboard button so the user can paste the result into the actual course feedback form.

- Store saved feedback locally in localStorage.
- Show a table/list of previously written feedback reviews.
- Include realistic demo examples:
  - “The best part was the clear problem framing and smooth demo flow.”
  - “One improvement would be to explain the LLM pipeline with one simple diagram.”
  - “The app idea is practical and the presentation clearly showed user value.”

6. AI / LLM Backend
Create backend API routes:
- /api/analyze
- /api/chat
- /api/feedback

/api/analyze should accept course text and return:
- summary
- extracted tasks
- deadlines
- missed/at-risk items
- weekly plan
- source breakdown

/api/chat should answer questions based on the current course data.

/api/feedback should accept rough peer feedback notes and return a polished review.

Support optional API keys:
- OPENAI_API_KEY
or
- ANTHROPIC_API_KEY

But the app must work even without API keys by using deterministic fallback demo outputs.
